简介
====================================

因为缺少测试预言，移动应用的功能测试是一个困难的问题。Kea根据PBT(基于性质的测试，Property
based Testing)理论进行测试，解决了脚本测试输入空间小，遍历测试无测试预言的问题。使应用可以在巨大的输入
空间下验证应用功能的正确性。Kea的输入是应用安装包及人工编写的“应用性质”，输出为测试报告。

在安卓系统的实验中，Kea在8个开源应用中发现了18个新功能性错误及7个回归错误。



Kea主要包含了：

(1) 一种基于 Python 的移动应用性质描述语言，帮助用户指定所需的应用性质。

(2) 三种探索策略，用于生成大量的 GUI 测试以验证这些性质。
